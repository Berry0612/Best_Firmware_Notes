## 重要概念
- volatile 是一個修飾字，用來告訴編譯器 (Compiler)：「這個變數隨時可能被外部因素（如硬體中斷、另一個執行緒、硬體暫存器）改變，所以請不要對它做最佳化 (Optimization)。」

- 當我們同時編譯多個檔案時，所有未加靜態的全域變數和函數都具有全域可見性。

- 當我在引入另外檔案的全域變數時，我需要在檔案開頭extern int a宣告，告訴編譯器說:這個變數去別的檔案找，如果我不宣告，編譯器會報錯，那假如我在檔案重新int 相同名稱的全域變數時，因為未使用 static 的全域變數具有「外部連結性（external linkage），linker 在合併時看到兩個相同的 symbol，就會報錯。
  
- static 最主要的作用是隱藏，當我static int a=10;，另外一個檔案extern int a; 會無法讀取，可以用在不同檔案

## 題目1： 
  假設有一個 8-bit 的暫存器 REG，目前數值未知。

1.請寫一段 C 語言程式碼，將 REG 的 第 3 個 bit (bit 3) 設定為 1 (Set bit)，同時不能影響到其他 bit。 

2.請寫一段 C 語言程式碼，將 REG 的 第 3 個 bit (bit 3) 設定為 0 (Clear bit)，同時不能影響到其他 bit。

下面是我的答案:

```C
#include <stdio.h>
int main(){
int set_bit3=4;
int ans_set,ans_clear ;

ans_set = (REG || set_bit3);
ans_clear = (REG && ~set_bit3);
}
```
### #錯誤檢討
1. 運算子用錯:|| 和 && 屬於邏輯運算子，當條件成立返回1(True)為非則返回0(False) --->要用位元運算子``| 和 & ``
2. 實際上不會用十進位來表示bit，如果今天要操作bit15，那麼2^15次方很難算 --->所以用位移運算子(Shift Operation):  `<<<`

來看正確解法:

1.Set Bit
```C
REG = REG | (1<<3); //可以簡寫成REG =| (1<<3)
```
2.Clear Bit
```C
REG &= ~(1<<3);
```

## 題目2：
  請閱讀以下 C 語言程式碼，告訴我最後 a 的值是多少？並解釋為什麼。
```c
#include <stdio.h>

void logic_test(int *p) {
    *p = *p + 5;
    p = p + 1; // 關鍵在這裡
    *p = *p + 5;
}

int main() {
    int a = 10;
    int b = 20;
    
    // 假設在記憶體中，b 剛好緊接在 a 的後面 (雖然現代編譯器不一定保證，但我們假設是連續的)
    // 呼叫函式
    logic_test(&a);
    
    // 請問現在 a 是多少？
    // 加分題：如果不考慮記憶體保護，b 的值會變嗎？
}
```

下面是我的答案:

a最後的值會是25。首先你假設在記憶體中b在a後面，logic_test(&a)會把 &a 放入 int*p 等於 int*p = &a，所以 *p 會指向 a 的值，p=a 的地址，

然後 *p = *p + 5; 這時 a=15 接著 p=p+1 可以看成 &a=&b  所以這時候 *p=b=20 接著 *p=*p+5 =25。

### #錯誤檢討
先講答案為a=15，b=25。
在p=p+1時，指標就從指向a轉成指向b，所以之後的操作跟a無關。
