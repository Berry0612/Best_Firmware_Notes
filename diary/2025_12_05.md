## 下面是Gemeni叔叔給我畫的大餅

### #專案一：自行實作周邊驅動 (Bare-Metal Peripheral Drivers)
目標： 不使用原廠 HAL 庫，純手工用 C 語言操作暫存器（就是我們剛剛考卷寫的那樣），寫出 GPIO, UART, I2C 的驅動程式。

應用： 讓 MPU6050 讀取數據，透過 I2C 傳給 STM32，再透過 UART 傳到電腦螢幕顯示，或者顯示在 OLED 上。

履歷亮點：

「熟悉 ARM Cortex-M 架構與暫存器操作」

「具備 I2C/UART 通訊協定底層實作能力」

「能夠閱讀 Datasheet 並轉化為驅動程式碼」

---

### #專案二：簡易排程器與即時作業系統 (Simple Scheduler / FreeRTOS)
目標： 在 STM32 上移植 FreeRTOS。

應用： 建立三個 Task：

Task A: 負責讀取感測器 (高優先級)。

Task B: 負責顯示到螢幕 (中優先級)。

Task C: 負責閃爍 LED (低優先級)。

並實作 Queue (佇列) 或 Semaphore (信號量) 來讓 Task 之間傳遞資料。

履歷亮點：

「熟悉 RTOS 多工原理與任務排程」

「了解 Race Condition 與 Inter-task Communication (IPC)」

---

### #專案三：韌體更新與 Bootloader (這是大絕招)
目標： 寫一個自定義的 Bootloader。

說明： 一般 MCU 上電就跑主程式。Bootloader 是在上電時先跑的一小段程式，它會檢查「有沒有要更新韌體？」，如果有，就從 UART 收資料把原本的程式覆蓋掉，然後再跳轉執行。

為什麼做這個？ 這展現了你對 Memory Map (記憶體配置)、Flash Programming (燒錄原理)、Function Pointer (函式指標) 的極致理解。
